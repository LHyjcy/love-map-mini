# PRD — love-map-mini

## 产品定位

情侣共同记录去过的地方、当时的故事和照片，同时用任务、积分、纪念日、打卡增强日常互动。
一个**私密的**情侣双人小程序，不是陌生人社交或公开社区。

## 核心闭环

1. **地图回忆闭环**：一起去某地 → 打卡 → 上传照片 → 写故事 → 地图点亮 → 周年回顾。
2. **任务互动闭环**：发布任务 → 对方完成 → 确认 → 获得积分 → 兑换情侣券 → 生成新回忆。
3. **约会计划闭环**：添加想去地点 → 设为计划 → 到达打卡 → 变成共同回忆 → 路线沉淀。

## MVP 范围

第一版必须做：微信登录、情侣绑定、地图地点、回忆故事、图片元数据、主动打卡、任务、积分、商城、日程、隐私设置。

第一版暂不做：后台持续定位、陌生人社交、公开社区、复杂支付、AI 生成日记、多情侣关系、实时轨迹回放。

## 地图模块（双地图 + 位置共享）

借鉴 map-of-us-template 的「省/市点亮」思路（不复制其源码/数据/品牌），将地图改造为双地图结构：

- **真实地图**（`pages/map/real`）：微信原生 map，展示地点/回忆/打卡 marker，点击弹底部卡片；入口：加地点、打卡、回忆、位置共享。
- **足迹地图**（`pages/map/footprint` + `/province` + `/city`）：canvas 绘制中国省/市边界（DataV GeoJSON，后端 `/api/geo/*` 下发），按情侣的地点/回忆点亮城市与省份；全国页展示点亮省/市/地点/回忆数；省→市→城市回忆逐级下钻。
- **地图首页**（`pages/map/index`，tabBar 入口）：四项足迹统计 + 真实地图/足迹地图/位置共享三入口。
- **位置共享**（`pages/location`）：仅用户主动开启的临时共享，强制过期、可随时停止；展示双方 marker、TA 最近更新时间、前端 Haversine 直线距离；WS 实时、断开降级轮询。详见 PRIVACY.md。
- **MapProvider 适配层**：腾讯（默认）/高德/百度（预留），key 仅后端；逆地址带缓存、搜索去抖、位置更新不触发逆地址。

足迹点亮规则：某城市存在「已去过地点」或「关联回忆的地点」即点亮该城市，其所属省份随之点亮（adcode 归一化）。
