# 产品功能规划与现状对照（PRODUCT_FUNCTION_PLAN）

> 依据《情侣地图产品功能分析与完整规划文档 v1.0》整理。产品定位：把「想去的地方、约会计划、
> 打卡、照片故事、纪念日、互动任务」沉淀成一张会成长的爱情地图。核心闭环：
> **想去 → 计划 → 打卡 → 照片 → 回忆 → 回顾 → 分享**。
> 边界：地图不是监督工具；位置共享默认关闭；公开地图默认关闭；不做后台持续定位。

## 一、核心闭环与现状（已基本打通）
| 闭环 | 流程 | 现状 |
|---|---|---|
| 地图回忆 | 去某地→打卡→上传照片→写故事→生成地图点→时间轴回看 | ✅ 已实现（真实地图 marker / 打卡 / 回忆+照片 / 时间轴） |
| 约会计划 | 想去→设为计划→到达打卡→一键转回忆 | ✅ 已实现（想去清单 / planAdd 设为计划 / 计划转回忆） |
| 互动激励 | 发任务→确认→积分→兑换情侣券→新回忆 | ✅ 已实现（任务状态机 / 积分流水 / 商城 / 背包） |

## 三、文档建议的优化 / 待补（按价值排序）
1. **底部导航重构（文档 §8）**：✅ 已实现。tabBar 重构为 `首页 / 地图 / 计划 / 互动 / 我的`（原 `首页 / 地图 / 任务 / 商城 / 我的`）。
   - 「计划」Tab = hub 页 `pages/plan/index`（想去清单 + 约会计划 + 倒计时·纪念日 + 计划转回忆聚合）。
   - 「互动」Tab = hub 页 `pages/interact/index`（任务 + 积分 + 商城 + 背包 + 今日心情 + 情侣问答聚合）。
   - tasks/shop 已降为二级页（switchTab→navigateTo）；新增 `pages/mood/index`、`pages/qa/index`。
2. **首页快捷入口（文档 §8.1）**：打卡 / 写回忆 / 发任务 / 添加计划 / 加地点 + 今日签到。✅ 本轮已补。
3. **轻量情感互动（差异化，文档 §4.5/§6.4）**：✅ 已实现。今日心情（Mood，每人每天一条、即时互看）、情侣问答（QaAnswer，**双方都回答后才互看**）。
   - 后端：新增 Mood / QaAnswer 模型 + `POST/GET /api/moods`、`GET/POST /api/qa/today`（详见 docs/API.md「情感互动」、docs/DATABASE.md）。
   - 前端：`pages/mood/index`、`pages/qa/index`，并聚合入「互动」hub。
4. **回忆标签系统（文档 §5.3 P0）**：✅ 已实现。Memory 新增 `tags`（逗号拼接，≤8 个、每个≤20 字符），
   `POST/PATCH /api/memories` 接受 `tags: string[]`，`GET /api/memories?tag=` 按标签模糊筛选。
5. **月度 / 年度回顾（文档 §4.3 / §6.3）**：✅ 已实现（后端）。`GET /api/review?period=month|year&value=`
   聚合回忆/地点/城市/省份/照片数 + 标签 Top5 + 近期回忆；分享海报（canvas 导出）留待前端补充。
6. **地点投票（文档 §4.1 P1）**：✅ 已实现。`POST /api/places/:id/vote`（want/meh/no，可改票）、
   `GET /api/places/:id/votes` 返回票数汇总 `tally`（含本人投票）。
7. **订阅消息真实下发**：✅ 真实下发已接（需配置微信模板/AppSecret 后生效）。`/subscribe` 落库
   `ReminderSubscription`，`/run-due` 到期经微信 `subscribeMessage` 下发；未配置时安全降级为 no-op。
8. **后台能力 / 运营底座（文档 §9）**：✅ 错误日志、访问限流已实现（图片清理 / 操作审计待补）。
   - 后端：错误处理器对 5xx **即发即忘**落库 `ErrorLog`（method/url/statusCode/code/message/userId）；
     `/api/auth/*`、`/api/couples/invite|accept` 启用**内存限流**（60s/30 次→`429 RATE_LIMITED`）。**均无外部依赖。**
9. **立体感地图体验（回忆直显地图 + 立体卡片 UI）**：✅ 已实现。回忆的照片+文字直接呈现在足迹/真实地图上，
   不再只是裸 marker。
   - 后端：`GET /api/footprint/cities/:cityId` 的 `memories[]` 富化，补 `content/mood/tags` + `photos[]`（媒体 fileUrl）
     + `cover`（首图），供前端取封面与文案（详见 docs/API.md「足迹地图」）。
   - 前端「立体感」：足迹城市页改为 3D「回忆相册」（带景深的 swiper 卡片：封面照片 + 文案 + 标签）；
     真实地图 marker 点击弹出「立体」底部卡（该地点封面照片 + 最新回忆文字）；memoryDetail 增加视差封面头图；
     点亮的省/市在足迹地图上点击先弹「立体」预览卡再下钻；新增「回忆墙」页（`pages/memoryWall/index`），
     以景深画廊形式陈列全部回忆。
10. **AI 文案（差异化，文档 §4.5 / §6.4）**：✅ 已实现。`POST /api/ai/memory-copy`（回忆标题+故事）、
    `/api/ai/share-caption`（分享配文）、`/api/ai/review-summary`（回顾小结），走 OpenAI 兼容 LLM；
    **未配置 AI Key 时自动回落本地模板**（`source` 标注 `llm`/`template`，无外部调用），Key 仅后端。
    前端：回忆创建表单与回顾页接入 AI 文案。
11. **Love Map 档案（差异化，文档 §6.3）**：✅ 已实现。`GET /api/profile`（双方小事实 + 双方都答的问答历史）、
    `POST /api/profile/fact`、`DELETE /api/profile/fact/:id`；新增 `ProfileFact` 模型与 `pages/profile/index` 页。
12. **用户反馈（运营，文档 §9）**：✅ 已实现。`POST /api/feedback`、`GET /api/feedback`（仅本人）；
    新增 `Feedback` 模型与 `pages/feedback/index` 页（未绑定情侣也可提交）。

## 四、隐私底线（文档 §10，现状均满足）
位置默认关闭、临时共享带过期、伴侣仅见主动分享且未过期位置、可删除历史、公开默认关闭+坐标模糊、
私有接口校验 userId+coupleId、密钥仅后端、解绑后权限隔离、不做后台持续定位。✅
