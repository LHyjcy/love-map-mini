# PRIVACY — love-map-mini

本应用涉及情侣关系数据、照片和位置，采用**隐私优先**的默认设置。

## 核心原则

- **不做后台持续定位**。位置仅在用户主动打卡或开启临时共享时采集。
- **位置共享默认关闭**。伴侣只有在用户明确选择分享后才能看到所选位置。
- **打卡仅由用户主动发起**，且临时共享可设置过期时间。
- **公开地图默认关闭**，且不默认暴露精确的家、学校、工作坐标。
- 用户可删除打卡、回忆和照片元数据。
- 每个私有接口都必须校验 `userId` 与 `coupleId` 访问权限。

## 数据删除

用户可删除：打卡（checkin）、回忆（memory）、照片元数据（media）。删除采用软删除，
后续可提供彻底清除入口。

## 照片采集（用户主动选择）

- 小程序的「为回忆添加照片」流程使用**用户主动发起**的 `wx.chooseMedia` 由用户**手动逐张选择**，
  **不读取、不遍历相册**，也不做任何后台自动取图。仅**用户选中的照片**会被上传。
- 照片元数据可随时删除（见下「数据删除」），删除采用软删除。

## 照片存储与上传

- **客户端直传，API 不中转文件字节（cos/oss 模式）**：照片由客户端经短时**预签名 URL** 直接 `PUT`
  上传到对象存储（腾讯云 COS / 阿里云 OSS），后端只签发凭证（`POST /api/media/upload-credential`）
  并登记元数据（`POST /api/media`），**原始图片字节不经过 API 服务器**，减少明文照片在服务端的
  留存与转发面。
- **预签名 URL 短时有效**：`cos`/`oss` 模式下 `uploadUrl` 为约 **10 分钟**有效的预签名 PUT 地址，过期作废、
  需重新签发，降低凭证被截留滥用的风险。`local` 开发模式不下发直传地址（`uploadUrl=null`）。
- **disk 模式（私用自托管）的例外与边界**：`STORAGE_PROVIDER=disk` 时照片字节经短时 HMAC 签名的
  `PUT /api/media/upload` 写入 **API 服务器本地磁盘**（即"字节不经过 API"的承诺仅适用于 cos/oss 模式）。
  读取走 `GET /files/<objectKey>`：objectKey 为 16 字节随机数（不可猜测的能力 URL），响应
  `Cache-Control: private`（不进共享缓存）+ `X-Content-Type-Options: nosniff`；**已删除照片立即 404**。
  删除照片元数据时服务端**同时尽力删除磁盘文件**（真删除，而非仅软删元数据）。上传签名密钥优先用
  `STORAGE_ACCESS_KEY_SECRET`，未设置时由 `JWT_SECRET` 派生**专用密钥**（不直接复用登录密钥）。
- **存储密钥仅在服务端环境变量**：`STORAGE_ACCESS_KEY_ID` / `STORAGE_ACCESS_KEY_SECRET` 仅从
  服务器环境变量读取，**绝不写入源码、绝不返回前端、绝不写入日志**；签名在服务端完成，客户端只拿到
  一次性的预签名地址。符合 CLAUDE.md「源码中不得包含任何真实密钥」的要求。
- **仅上传用户选择的照片**：上传内容仅限用户通过 `wx.chooseMedia` 主动选中的图片，限图片类型
  （jpg/png/webp/gif）、单张 ≤ 10MB。
- **元数据可删除**：照片元数据可随时删除（软删除，见「数据删除」），删除后不再参与任何查询/展示。

## 内容安全审核（平台合规的数据流披露）

- 为满足微信小程序平台的 UGC 合规要求，**回忆的标题与正文**（`POST/PATCH /api/memories`，含
  `visibility=private/couple` 的私密回忆）、**公开分享标题**（`POST /api/public-shares`）与
  **意见反馈文本**（`POST /api/feedback`）在保存前会提交到**微信官方内容安全接口
  `msg_sec_check`**（`api.weixin.qq.com`）做合规检查。
- 该数据流的边界：仅在**用户主动提交**时发送被提交的文本本身；本应用**不存储**微信侧的审核
  原始结果，仅据其放行或返回 `400 CONTENT_REJECTED`。
- 服务端未配置微信 AppID/Secret 时（如本地开发、纯自托管），审核**直接放行（fail-open）**，
  文本**不会离开服务器**。

## 敏感地点

建议用户不要把家、学校、工作等敏感地点设为公开可见。公开地图坐标默认做轻微模糊
（保留 3 位小数）。

## 认证与最小化（Phase 4）

- 登录令牌为 JWT，载荷**仅含用户 id（`sub`）**，不放昵称、openid 等可识别信息。
- `JWT_SECRET` 只从环境变量读取；生产缺失**或为占位/过短（<16 位）值**即启动失败（fail-closed），
  绝不回退占位值，源码不含真实密钥。docker-compose 同样不再提供不安全默认值，必须显式设置。
- mock 登录默认**仅限开发**，生产环境直接拒绝（`MOCK_LOGIN_DISABLED`）；私用自托管确需
  「体验登录+邀请码绑定」时，须显式设置 `MOCK_LOGIN_ENABLED=true`（明确的 opt-in，不隐式依赖环境）。
- **注销后令牌立即失效**：鉴权层在校验 JWT 签名后还会确认用户**存在且未注销**，
  `POST /api/account/delete` 之后旧令牌一律 `401`，不存在"注销后令牌仍可用 7 天"的窗口。
- 微信登录走**服务端 `code2session`**（生产登录路径）：`WECHAT_APP_ID/SECRET` 仅从服务端环境变量
  读取，绝不写死、绝不返回前端；`code2session` 换回的 `session_key` **仅服务端短暂使用，绝不返回
  前端、绝不写入可读日志**；仅持久化 `openid`（及微信下发的 `unionid`），且**不向客户端暴露**。
  服务端未配置 AppID/Secret 时（开发环境）该接口直接返回 `501 WECHAT_NOT_CONFIGURED`，不会泄露密钥。
- 用户/伴侣对外仅暴露最小字段（id、昵称、头像、性别），不返回 openid/unionid/session_key。
- 邀请码为 6 位数字、24 小时过期、绑定后失效；除按 IP 的接口限流外，**接受邀请另有按用户的
  失败锁定**（1 小时内失败 10 次即 `429 TOO_MANY_ATTEMPTS`），抵御分布式暴力猜码。

## 位置打卡与共享（Phase 6）

- 打卡（`POST /api/checkins`）**仅用户主动发起**，服务端不做任何后台/持续定位。
- 打卡 `shareScope` 默认 `self`（仅本人可见）。只有显式选择 `partner`/`memory` 才可能被伴侣看到。
- 临时共享可设 `shareTtlMinutes` → `expiresAt`；**`shareScope='partner'` 未指定有效期时服务端
  默认 120 分钟过期**，杜绝"无限期可见"的位置共享；`memory`（保存为共同回忆）为显式的永久动作，
  不套默认过期。`GET /api/checkins/partner-latest` 只返回伴侣 `shareScope ∈ {partner, memory}`
  **且未过期**的最近一条，过期即不可见。
- 打卡可软删除（`DELETE /api/checkins/:id`），删除后不再参与任何查询。

## 隐私授权记录（Phase 10）

- `PrivacyConsent` 按用户记录 `location/album/camera/public_share` 的同意与撤销，**追加写**、
  可审计，不就地修改历史。当前状态取每种类型的**最近一条记录**（`agreedAt` 有值即为已同意）。
- 小程序「隐私中心」`pages/privacy/privacy` 提供上述四类授权的开关，调用
  `GET /api/privacy/consents` 读取当前状态、`POST /api/privacy/consents` 写入同意/撤销。
- 所有授权开关**默认关闭（OFF）**；用户可**随时撤销**任意一项，撤销同样以追加记录方式留痕，
  不删除历史，便于审计。
- **行为即授权**：用户主动发起共享类动作（向伴侣共享打卡、开启临时位置共享 → `location`；
  显式开启公开地图 → `public_share`）时，服务端自动补记一条同意记录（`version=action-implied-v1`），
  保证授权台账与实际行为一致；最新状态已是同意时不重复写。
- **撤销即生效**：撤销 `location` 会**立即停止**本人所有生效中的临时位置共享会话并清掉伴侣可见的
  最新位置缓存；撤销 `public_share` 会**立即关闭**本情侣所有已开启的公开地图分享——开关不是
  只记台账的摆设。

## 公开地图与公开分享（Phase 10）

- 公开分享**默认关闭**。小程序「公开分享」`pages/publicShare/publicShare` 通过
  `/api/public-shares` 创建/列出/启用/停用公开分享记录。
- 文案明确告知：公开分享**仅展示公开内容**，且坐标会做**模糊处理**；
  默认仅 `visibility=public` 的数据可进入公开范围，家/学校/工作等精确坐标不外泄。
- **坐标模糊化已实现**：免登录的公开读取接口 `GET /api/public-map/:shareCode` 在返回前
  将所有经纬度统一保留 **3 位小数（约 110m）**，并仅返回 `visibility=public` 的地点与回忆，
  不返回 openid、详细地址、定位精度等任何超出约定的字段。分享停用或不存在一律 `404`。
- web-share 公开页（Leaflet）**只读取并渲染上述接口返回的已模糊化公开内容**，
  自身不持有也无法获取精确坐标，**精确的家/学校/工作坐标在任何路径下都不会被暴露**。
- 用户可**随时停用**任一公开分享链接（disable），停用后该链接立即返回 `404`、不再对外可见。

## 同行打卡提示（前端启发式，无追踪）

- 「打卡」页在伴侣距离 **≤ 200m** 时显示「同行打卡」提示。该提示是**纯前端启发式**，
  基于页面**已获取的数据**计算距离，**不做后台追踪、不发起额外定位**，
  与「不做后台持续定位」「打卡仅由用户主动发起」原则一致。

> 对象存储签名直传已落地（见「照片存储与上传」）：上传密钥仅在服务端环境变量、不进前端、不入日志。
> 微信登录的 `code2session` 走服务端 AppID/AppSecret，`session_key` 同样仅服务端短暂使用、
> 不返回前端、不入可读日志；仅 `openid`/`unionid` 入库且不对客户端暴露。
> 两者均满足 CLAUDE.md「源码中不得包含任何真实密钥」的要求。

## 共同打卡判定（服务端，仅读既有打卡，不外泄坐标）

- 共同打卡判定接口 `GET /api/co-checkin` 在**服务端**依据双方**已存在的、用户主动发起的**
  最近若干条打卡计算距离与时间差，仅返回**粗粒度距离（米）**、**时间差（分钟）**、双方各自的
  `checkinId`，以及当打卡关联地点时该地点的 `placeId/placeTitle`。
- **伴侣侧仅匹配已共享、未过期的打卡**：`shareScope='self'`（默认）或已过期的打卡**不参与匹配**，
  防止私密打卡通过"相距 X 米 / 相隔 X 分钟"被旁路推断出来。
- **本接口只读取既有打卡记录，不做后台持续定位，不发起任何额外定位或上报**；
  与「打卡仅由用户主动发起」「不做后台持续定位」原则一致。
- **响应绝不返回伴侣的原始经纬度或身份标识**，避免借此反推伴侣的精确位置；
  与「伴侣位置须显式同意」原则一致。无候选时返回 `candidate: null`。

## 数据导出（仅限本情侣，呼应解绑后数据归属）

- `GET /api/export` 允许用户把**属于自己情侣关系（coupleId）**的全量数据**完整复制一份**带走
  （备份/数据可携），**绝不跨情侣**。
- 导出内容**不包含任何 `openid/unionid/sessionKey`** 等身份敏感字段，也**不含任何应用密钥**；
  双方用户摘要仅含 `id/昵称/头像`，隐私授权记录按当前 `userId` 维度导出。
- 该能力让用户可随时导出/留存自己的数据，呼应「提前定义解绑后数据可见性与导出规则」的要求。

## 订阅消息提醒（用户授权 opt-in，当前占位）

- 提醒类订阅消息（纪念日/计划提醒）在小程序侧由 `wx.requestSubscribeMessage` 触发，
  **必须经用户主动授权（opt-in、知情同意）后**才会调用 `POST /api/notifications/subscribe`。
- 当前为**占位实现**：服务端仅校验入参，**不调用微信、不落库、不产生任何真实推送**，
  因此不会在用户未察觉的情况下发送消息。真实下发为后续阶段，届时仍以用户授权为前提。

## 临时位置共享（地图模块核心隐私约束）

- **不做默认后台持续定位**：仅使用 `wx.startLocationUpdate`（前台），**不使用** `wx.startLocationUpdateBackground`，
  不存在监控式/后台持续追踪。
- **仅用户主动开启**：共享需用户点击「共享 30 分钟 / 2 小时」显式开启，会话强制带 `expiresAt`。
- **可随时停止 + 自动过期**：用户可手动停止；到期后服务端将会话置 `expired`，最新位置缓存（Redis/内存）随 TTL 失效。
- **伴侣可见性**：伴侣只能看到对方 **active 且未过期** 会话的**最新一个**位置点；停止/过期后立即不可见
  （WS 推送 `partner_location_stopped/expired`，轮询接口也不再返回）。
- **不保存高频全量轨迹**：上传经节流（间隔≥10s、移动≥30m、精度>100m 不传），`LocationPoint` 仅留节流样本，可软删除。
- **坐标系**：默认 GCJ-02；位置更新**不触发逆地址解析**（逆地址仅在地点/搜索等用户操作时按需调用并缓存）。
- **地图服务商不参与实时位置同步**：仅用于逆地址/搜索/路线；地图 key 仅后端环境变量，**绝不写入小程序前端、不记日志**。
- **越权隔离**：位置相关私有接口全部 `requireAuth + coupleId` 校验；WebSocket 连接需 JWT，仅加入本人 `couple` 房间、仅推送给伴侣。
